## 🔍 Overview

A **Flash ADC** (also called a *parallel ADC*) is the fastest Analog-to-Digital Converter architecture available, since it performs conversion in a **single clock cycle** using parallel comparison rather than iterative approximation.

This project implements a **2-bit Flash ADC** at the transistor (CMOS) level using the **Microwind & DSCH** design suite, covering the complete flow from schematic → comparator design → layout → post-layout simulation.

A 2-bit Flash ADC requires:

| Component | Count | Purpose |
|---|---|---|
| Resistor Divider Network | 4 × R | Generates reference voltages |
| Comparators | 3 | Compares `Vin` against each reference level |
| Priority Encoder | 1 | Converts thermometer code → 2-bit binary |

> **Why Flash ADC?** With *N* comparators operating simultaneously (not sequentially), an *n*-bit Flash ADC achieves conversion in ~1 clock cycle — at the cost of requiring `2ⁿ − 1` comparators, which limits scalability for higher resolutions.

The analog input `Vin` is fed simultaneously to the negative terminal of all 3 comparators. Each comparator's positive terminal is biased at a fixed fraction of `VREF`, generated by a resistor-divider ladder. The three parallel comparisons happen in one shot — hence "Flash".

## 🧮 Circuit Design

### 1. Voltage Reference Network

A simple resistor ladder (4 × R, VREF = 2V) generates 3 evenly spaced reference voltages:

$$V_1 = \frac{V_{ref}}{4} = 0.5V \qquad V_2 = \frac{V_{ref}}{2} = 1.0V \qquad V_3 = \frac{3V_{ref}}{4} = 1.5V$$

This converts the continuous analog wiper-position signal into discrete step levels — the foundation of quantization.

### 2. Comparator Stage

Each comparator compares `Vin` against its assigned reference voltage:

- `Vin > Vref` → Output **LOW** (inverting comparator convention used here)
- `Vin < Vref` → Output **HIGH**

At the transistor level, each comparator is a **9-transistor differential-input CMOS comparator** — differential pair (M1, M2), PMOS current-mirror load (M3, M4), and a tail/bias network (M5–M9):

### 3. Thermometer Code

The 3 comparator outputs (`C1 C2 C3`) form a **thermometer code** — bits turn on sequentially as `Vin` rises:

| Vin Range | C1 C2 C3 |
|---|---|
| Vin < V1 | `000` |
| V1 < Vin < V2 | `100` |
| V2 < Vin < V3 | `110` |
| Vin > V3 | `111` |

### 4. Priority Encoder

The thermometer code is compressed into 2-bit binary using combinational logic:

<p align="center">
  <img width="509" height="368" alt="image" src="https://github.com/user-attachments/assets/3fe5a572-9cd6-4098-8133-0c0479e47fc0" />

</p>

| Thermometer | Binary (Q1 Q0) |
|---|---|
| `000` | `00` |
| `100` | `01` |
| `110` | `10` |
| `111` | `11` |

**Boolean Equations:**

```
Q1 (MSB) = C1 + C2
Q0 (LSB) = C1 + (¬C2 · C3)
```

**Gate-level realization:**

<p align="center">
  <img src="images/06_encoder_gate_logic.png" alt="Priority Encoder Gate-Level Logic" width="700">
</p>

| Gate | Inputs | Expression | Role |
|---|---|---|---|
| NOR₁ | C1, C2 | ¬(C1 + C2) | First stage of OR (MSB path) |
| NOT₁ | ¬(C1+C2) | C1 + C2 | Produces MSB (Q1) |
| NOT₂ | C2 | ¬C2 | Invert C2 |
| NAND | ¬C2, C3 | ¬(¬C2 · C3) | First stage of AND |
| NOT₃ | ¬(¬C2·C3) | ¬C2 · C3 | AND of ¬C2 and C3 |
| NOR₂ | C1, ¬C2·C3 | ¬(C1 + ¬C2·C3) | First stage of OR (LSB path) |
| NOT₄ | ¬(C1+¬C2·C3) | C1 + ¬C2·C3 | Produces LSB (Q0) |

## 📊 Simulation Results

**Test conditions:**
- **Input:** Analog triangular/ramp waveform sweeping 0V → VREF
- **Output:** 2-bit digital binary code, correctly stepping through `00 → 01 → 10 → 11` as input rises

<p align="center">
 <img width="659" height="356" alt="image" src="https://github.com/user-attachments/assets/8198903f-136a-482a-ae29-d52ede911f33" />

</p>

The transient simulation confirms:
- Comparator outputs (`c1`, `c2`) switch cleanly at their respective threshold crossings
- Encoder outputs (`o1`, `o2`) track the expected binary transitions with propagation delays in the sub-nanosecond range (~600–700 ps)
